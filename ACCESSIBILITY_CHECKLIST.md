# Accessibility Checklist for Atomize

## Core improvements to aim for
- Ensure all interactive elements are keyboard accessible.
- Provide clear focus states for links, buttons, and form fields.
- Maintain sufficient color contrast for text and controls.
- Use semantic headings and landmarks for screen readers.
- Add meaningful labels to form inputs and buttons.
- Ensure forms and dialogs are readable and navigable.
- Provide alt text for meaningful images.
- Keep text readable with scalable font sizes.
- Make the app usable on mobile and desktop without relying on touch-only gestures.

## Recommended next steps
1. Review the web app with keyboard-only navigation.
2. Test with screen readers such as NVDA or VoiceOver.
3. Confirm accessibility contrast ratios on buttons and text.
4. Add an accessibility statement page if you want to present your compliance intentions clearly.
